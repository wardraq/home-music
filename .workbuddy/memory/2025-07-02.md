## NCM 解密测试 + bug 修复

wardraq 要求用本地网易云音乐文件测试 home-music 项目。

### 测试文件
- `/Users/yu.xiao/Music/网易云音乐/Alex Warren - Save You a Seat.ncm`（7.9MB）

### 发现并修复的 bug

**Bug 1: artist 字段解析崩溃**
- 错误：`'list' object has no attribute 'get'`
- 原因：NCM 元数据 artist 字段实际格式是 `[["Alex Warren", "29631357"]]`（list of lists），代码假设是 list of dicts
- 修复：`_parse_meta_json` 兼容 dict / list / str 三种 artist 元素格式

**Bug 2: 音频数据偏移错误**
- 错误：解密输出是乱码，`file` 识别为 `data` 而非 `Audio file`
- 原因：文件结构解析有误
  - 原代码：`f.read(4)` (CRC32) + `f.seek(5,1)` (gap) = 9 bytes，实际应为 5 bytes
  - 原代码漏读 `image_space` 字段，不跳过封面填充区（本文件 image_space=7550, image_size=0）
  - 导致音频从 pos 817 开始读，实际应从 pos 8367 开始
- 修复：`f.seek(5,1)` 跳过 CRC32+gap，然后读 image_space + image_size，seek 跳过填充

### 验证结果
- 解密输出：`Save You a Seat.mp3`，`file` 确认为 MPEG ADTS, layer III, v1, 320 kbps, 48 kHz, Stereo
- ID3 标签：TIT2=Save You a Seat, TPE1=Alex Warren, TALB=Save You a Seat
- 时长 197.6s，比特率 320kbps，与 NCM 元数据一致
- organize.py 正确分类到 `Alex Warren/Save You a Seat/Save You a Seat.mp3`

## 一生所爱 FLAC → ALAC 转换 (2025-07-02)

wardraq 反馈 Apple Music 无法导入 FLAC，需要转换。

### 文件信息
- `/Users/yu.xiao/Music/网易云音乐/卢冠廷,AGA - 一生所爱.ncm`
- 解密为 FLAC：96kHz / 24bit / 122.9MB
- Apple Music 不支持 FLAC，需转 ALAC

### 转换方案
- 使用 macOS 自带 `afconvert`：`afconvert -f m4af -d alac input.flac output.m4a`
- afconvert 不保留 FLAC 的 Vorbis 标签，用 mutagen 手动复制 title/artist/album
- 转换后：96kHz / 24bit / ALAC / 178MB，音质无损

### 修复 organize.py 的 m4a 标签读取
- 原代码用 `MutagenFile()` 读 m4a 时，MP4 标签键为 `\xa9nam/\xa9ART/\xa9alb`，原逻辑只匹配 `"title"`/`"artist"`/`"album"` 字符串，导致 m4a 被分类到 `未知歌手/未知专辑`
- 修复：增加 MP4 标签键映射，并正确处理标签值为列表/元组的情况

### 最终结果
- `music_library/卢冠廷、AGA/最经典的演唱会 Live/一生所爱.m4a`
- 标签正确：歌名/歌手/专辑
- 可直接拖入 Apple Music

### 依赖
- Python venv: `/Users/yu.xiao/.workbuddy/binaries/python/envs/default`
- 包：mutagen, pycryptodome, pyyaml

## NCM 解密密钥外部化 (2025-07-02)

wardraq 反馈安全软件（马维斯）提示风险，要求将密钥放到本地配置并排除在仓库外。

### 调整
- 新增 `config/ncm_keys.yaml` 存放 NCM 解密所需的 AES 密钥（core_key / meta_key / magic）
- 新增 `config/ncm_keys.yaml.example` 作为模板，仓库内只提交模板
- `.gitignore` 增加 `config/ncm_keys.yaml`、`config/*.local.yaml`、`.env`
- `ncm_decrypt.py` 启动时从 `config/ncm_keys.yaml` 加载密钥，不再硬编码在代码里
- 如果配置文件缺失，脚本提示用户复制 example 文件

### 说明
这些密钥是 NCM 公开文件格式常量，并非用户敏感凭证。移到配置文件中主要是为了：
1. 减少安全软件对硬编码密钥的误报
2. 未来若密钥变化或需要多组密钥，无需改代码
3. 仓库代码中不再出现任何看起来像密钥的字符串

### 验证
- 解密测试通过（`Alex Warren - Save You a Seat.ncm`）
- `git status` 确认 `config/ncm_keys.yaml` 已被忽略，未出现在变更列表中
