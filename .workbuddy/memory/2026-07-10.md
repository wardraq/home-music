# Home Music 工作日志 - 2026-07-10

## 继续修复 CODE_REVIEW.md 遗留问题

本次处理了代码审查报告中尚未修复的问题。报告本身状态已偏旧（只显示 4 项已修复），实际此前已修复更多问题。本次又完成 9 项新修复：

### 已修复问题

1. **#1 watcher.py 竞态条件**
   - 在 `NcmHandler` 中添加 `self._processing` 集合 + `threading.Lock`。
   - `on_created` / `on_modified` 同时触发同一文件时，第二次直接返回，避免重复处理。

2. **#2 跨文件系统 move 风险**
   - `scripts/watcher.py` 增加 `safe_move()`：显式 `shutil.copy2()` + `os.remove()`，兼容跨文件系统。
   - `scripts/organize.py` 的 `move=True` 也改用 `_safe_move()`，统一行为。

3. **#3 observer.join() 无超时**
   - `observer.join(timeout=3600)`，超时后记录错误并强制继续退出流程。

4. **#11 30 秒超时无重试**
   - 增加后台重试线程 + `_retry_queue`。
   - 文件等待超时时加入队列，最多重试 `_MAX_RETRY_ATTEMPTS=3` 次，超出则标记为死信日志。
   - `_wait_for_file_ready` 增加连续两次大小稳定才认为就绪，减少抖动误判。

5. **#12 临时文件清理不完善**
   - `_do_process_file` 用 `try/finally` 包裹，解密后的临时文件在成功/失败/异常时都会被清理。

6. **#13/#27 import 与路径问题**
   - `hashlib` 从函数内部移到 `watcher.py` 和 `utils.py` 的模块顶部。
   - `watcher.py` 的 `sys.path.insert` 改为显式 `Path(__file__).resolve().parent`，并检查是否已存在，避免重复插入。
   - 新增 `scripts/__init__.py`，为 `pyproject.toml` 包发现做准备。

7. **#9 organize_file 返回值语义不清**
   - 新增 `OrganizeResult` 数据类，状态区分 `success` / `failed` / `unsupported` / `error`。
   - 更新 `organize_directory` 和 `main()` 的统计逻辑，分别输出成功/失败/不支持/错误数量。
   - `watcher.py` 调用处改用 `organized.path` / `organized.message`。

8. **#10 processed.json 无限增长**
   - `load_processed` / `save_processed` 中调用 `_expire_processed()`。
   - 过期策略：保留最近 `MAX_PROCESSED_AGE_DAYS=90` 天记录，同时限制最多 `MAX_PROCESSED_ENTRIES=10000` 条。

9. **#19 专辑封面 MIME 类型仅 JPEG/PNG**
   - 新增 `_detect_image_mime()`，支持 JPEG、PNG、WebP、BMP、GIF 的魔数检测。

10. **#15 缺少 pyproject.toml**
    - 新增 `pyproject.toml`，声明项目元数据、依赖、可选 dev 依赖和 console scripts。
    - 依赖：mutagen、pycryptodome、watchdog、pyyaml；dev 依赖：pytest、pytest-cov。
