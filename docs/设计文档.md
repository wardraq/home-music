# Home Music 设计文档

## 一、产品定位

**它是什么：**
将网易云音乐下载的加密 .ncm 文件解密为原始音频，自动整理分类，通过 WiFi 同步到手机离线播放。

**它不是什么：**
- 不是在线流媒体服务器（不需要 24 小时跑服务）
- 不是音乐播放器（用 VLC 等现成播放器）
- 不是盗版工具（仅处理用户已下载/已购买的歌曲）

**核心场景：**
1. 网易云下载 .ncm → 自动解密为 MP3/FLAC
2. 自动补全标签、嵌入封面、按歌手/专辑分类
3. 手机连同一 WiFi，从电脑同步音乐到手机离线听

---

## 二、技术架构

```
网易云客户端
    ↓ 下载 .ncm 到本地目录
监听脚本 (watcher.py)
    ↓ 检测到新 .ncm 文件
解密引擎 (ncm_decrypt.py)
    ↓ .ncm → mp3/flac + 提取封面/歌词/标签
整理脚本 (organize.py)
    ↓ 补全 ID3 标签、重命名、分类
音乐库 (music_library/)
    ↓ 歌手/专辑/歌曲
同步服务 (VLC WiFi Upload 或自建)
    ↓
手机端 (VLC for iOS)
```

---

## 三、NCM 解密

### NCM 文件结构

```
Header: "CTENFDAM" (8 bytes)
Key Data: AES-128-ECB 加密的 RC4 密钥
Meta Data: AES-128-ECB 加密的 JSON（歌手、专辑、封面 URL 等）
Album Image: 专辑封面图片
Music Data: RC4 变异算法加密的音频数据
```

### 解密流程

1. 读取 Header 验证格式
2. 解密 RC4 密钥
3. 解密 Meta Data（JSON 格式，含歌曲信息）
4. 跳过封面图片
5. 用 RC4 解密音频数据
6. 输出为 MP3 或 FLAC（根据 Meta 中的格式字段）
7. 将封面、歌词写入 ID3 标签

## 四、自动整理

### 文件命名规则

```
music_library/
├── 周杰伦/
│   ├── 叶惠美/
│   │   ├── 01 - 以父之名.flac
│   │   ├── 02 - 懦夫.flac
│   │   └── ...
│   ├── 七里香/
│   │   └── ...
│   └── 单曲/
│       └── 稻香.flac
├── 林俊杰/
│   └── ...
└── 未知歌手/
    └── ...
```

### 元数据来源优先级

1. NCM 内嵌的 Meta Data（最准确）
2. 文件名解析（歌手 - 歌名.mp3）
3. 留空，标记为"未知"

### ID3 标签处理

- 歌名、歌手、专辑、年份、 Track Number
- 嵌入专辑封面
- 嵌入歌词（如有）

## 五、自动监听

### 监听目录

- macOS: `~/Music/CloudMusic/` 或用户自定义
- 检测到 `.ncm` 文件写入完成 → 自动解密 → 整理到 music_library

### 防重复处理

- 记录已解密文件的 MD5，重复文件跳过
- 记录日志：`config/processed.json`

## 六、手机同步

### 方案 A：VLC WiFi Upload（MVP，零开发）

1. 手机打开 VLC → WiFi Upload
2. 电脑浏览器打开手机显示的地址
3. 拖拽音乐文件进去
4. 手机 VLC 离线播放

### 方案 B：自建 HTTP 同步（未来迭代）

1. 电脑跑 HTTP 服务，暴露音乐库目录
2. 手机端 App 浏览 → 选歌 → 下载
3. 需要开发 iOS 客户端或用 Subsonic 兼容客户端

## 八、关键决策

1. **只用 Python，不写 App** — 解密和整理都是脚本，手机端用 VLC
2. **MVP 用 VLC WiFi Upload** — 零开发成本验证流程
3. **不搭流媒体服务器** — 先做离线同步，在线流媒体是 Phase 4
4. **Meta Data 从 NCM 内嵌提取** — 比文件名解析准确，不需要联网查
5. **music_library 不进 Git** — 音乐文件不入版本控制
